## 🔧 完整部署步骤

### 第一步：连接到服务器

```bash
# 使用 SSH 连接到阿里云服务器
ssh root@your-server-ip
```

### 第二步：上传部署文件（本地操作）

打开 PowerShell，在项目根目录执行：

```powershell
# 1. 上传 nginx 配置
scp d:\Doc\HBuilderProjects\ChiHai\nginx.conf root@your-server-ip:/tmp/

# 2. 上传部署脚本
scp d:\Doc\HBuilderProjects\ChiHai\deploy.sh root@your-server-ip:/tmp/
```

### 第三步：在服务器上运行部署脚本

```bash
# 1. 进入临时目录
cd /tmp

# 2. 给脚本执行权限
chmod +x deploy.sh

# 3. 运行部署脚本
sudo bash deploy.sh
```

### 第四步：上传前端文件（本地操作）

```powershell
# 上传 dist 文件夹内容到服务器
scp -r d:\Doc\HBuilderProjects\ChiHai\dist\* root@your-server-ip:/var/www/chihai/
```

### 第五步：上传后端文件（本地操作）

```powershell
# 创建后端目录并上传
# 先在服务器创建目录，在服务器上执行：
sudo mkdir -p /var/www/chihai-backend
sudo mkdir -p /var/www/chihai-backend/server/data
sudo mkdir -p /var/www/chihai-backend/server/uploads

# 然后本地上传后端文件
scp d:\Doc\HBuilderProjects\ChiHai\server\index.js root@your-server-ip:/var/www/chihai-backend/server/
scp d:\Doc\HBuilderProjects\ChiHai\server\package.json root@your-server-ip:/var/www/chihai-backend/server/
```

### 第六步：在服务器上配置后端

```bash
# 1. 进入后端目录
cd /var/www/chihai-backend/server

# 2. 如果没有安装 Node.js，先安装 Node.js
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 3. 安装依赖（如果有 package.json）
npm install

# 4. 确保数据目录存在
mkdir -p data uploads

# 5. 使用 PM2 启动后端服务
# 先安装 PM2
sudo npm install -g pm2

# 启动后端服务
pm2 start index.js --name chihai-backend

# 设置开机自启
pm2 startup
pm2 save

# 查看状态
pm2 status
```

### 第七步：设置文件权限

```bash
# 设置正确的文件权限
sudo chown -R www-data:www-data /var/www/chihai
sudo chmod -R 755 /var/www/chihai

# 设置后端目录权限
sudo chown -R $USER:$USER /var/www/chihai-backend
sudo chmod -R 755 /var/www/chihai-backend
```

### 第八步：验证部署

在浏览器访问：
```
http://your-server-ip
```

访问后端管理页面：
```
http://your-server-ip/admin
```

## 📁 文件传输方法

### 方法一：使用 SCP（命令行）

```powershell
# 上传单个文件
scp file.txt root@your-server-ip:/path/

# 上传整个文件夹
scp -r dist/ root@your-server-ip:/var/www/chihai/
```

### 方法二：使用 WinSCP（推荐，图形界面）

1. 下载 WinSCP：https://winscp.net/
2. 连接信息：
   - 协议：SFTP
   - 主机名：服务器 IP
   - 端口：22
   - 用户名：root
   - 密码：您的服务器密码
3. 左侧导航到：`d:\Doc\HBuilderProjects\ChiHai\dist`
4. 右侧导航到：`/var/www/chihai`
5. 拖拽所有文件上传

### 方法三：使用 FileZilla

1. 下载 FileZilla：https://filezilla-project.org/
2. 连接信息同 WinSCP
3. 上传 `dist` 文件夹内容到 `/var/www/chihai`

## 🔧 常用命令

### Nginx 管理

```bash
# 启动
sudo systemctl start nginx

# 停止
sudo systemctl stop nginx

# 重启
sudo systemctl restart nginx

# 重新加载配置
sudo systemctl reload nginx

# 查看状态
sudo systemctl status nginx

# 测试配置
sudo nginx -t

# 查看访问日志
sudo tail -f /var/log/nginx/access.log

# 查看错误日志
sudo tail -f /var/log/nginx/error.log
```

### 防火墙设置

```bash
# 检查防火墙状态
sudo ufw status

# 开放 80 端口（HTTP）
sudo ufw allow 80/tcp

# 开放 443 端口（HTTPS）
sudo ufw allow 443/tcp

# 启用防火墙
sudo ufw enable
```

## 🔒 配置 HTTPS（可选但推荐）

### 1. 安装 Certbot

```bash
sudo apt-get update
sudo apt-get install -y certbot python3-certbot-nginx
```

### 2. 获取 SSL 证书

```bash
sudo certbot --nginx -d your-domain.com -d www.your-domain.com
```

### 3. 自动续期

```bash
sudo certbot renew --dry-run
```

## 🚨 常见问题

### 问题 1：网站无法访问

**检查列表**：
```bash
# 检查 Nginx 状态
sudo systemctl status nginx

# 检查防火墙
sudo ufw status

# 检查端口监听
sudo netstat -tlnp | grep nginx

# 查看 Nginx 错误日志
sudo tail -f /var/log/nginx/error.log
```

### 问题 2：403 Forbidden 错误

```bash
# 检查文件权限
ls -la /var/www/chihai

# 修复权限
sudo chown -R www-data:www-data /var/www/chihai
sudo chmod -R 755 /var/www/chihai
```

### 问题 3：后端 API 无法访问

```bash
# 检查 PM2 状态
pm2 status

# 查看后端日志
pm2 logs chihai-backend

# 重启后端服务
pm2 restart chihai-backend
```

### 问题 4：图片无法显示

```bash
# 检查上传目录权限
ls -la /var/www/chihai-backend/server/uploads

# 确保后端有权限写入
sudo chown -R $USER:$USER /var/www/chihai-backend/server/uploads
sudo chmod -R 755 /var/www/chihai-backend/server/uploads
```

## 💡 提示

1. **备份数据**：部署前先备份现有文件和数据库
2. **测试环境**：先在测试环境验证后再生产部署
3. **监控日志**：定期检查 Nginx 和 PM2 日志
4. **安全加固**：配置防火墙，定期更新系统
5. **域名配置**：在阿里云控制台配置 DNS A 记录
